FAQ Accordion
6 Sections

Navigate to Employees → All Employees, find the record, click the action menu (⋮), and select Reset Credentials. A temporary password is emailed to the registered address. The employee must change it on first login. Admin access is required to perform this action.

Go to Employees → All Employees and use the filter panel to narrow your selection. Click Export → CSV or Export → Excel from the top-right action bar. Exports include all visible columns by default; use column visibility settings to customize the output.

Leave requests appear in Employees → Leave Requests with a Pending status badge. Department heads receive an email notification. They can approve or reject directly from the email link, or from the panel. All decisions are logged with timestamp and approver name for audit purposes.

A Critical Error notification appears in the bell dropdown and an alert email is sent to system administrators. Data entered during the outage is queued locally and synced automatically once connection is restored. No data is lost during brief outages of under 30 minutes.

Go to System → Permissions → Roles. Create a role, define module-level access (read / write / admin), and assign employees to it. Changes apply immediately without requiring a re-login. Super Admin role cannot be modified for security compliance.

Navigate to Settings → System → Storage Manager. Use the auto-archive tool to compress logs older than 90 days. You can also delete expired audit trails and temporary export files. Bulk cleanup typically recovers 15–30% of used space.
